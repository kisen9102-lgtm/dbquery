## 一、用户认证与权限系统

### 需求
- 添加登录 / 注册功能
- 用户分三种角色：`dbsroot`（超级用户）、`admin`（管理员）、`query`（查询用户）
- 默认注册为 query 角色，不能访问任何数据库，需加入用户组才行
- 用户组的添加和修改权限仅限 admin / dbsroot

### 实现内容

**新建 `accounts` app**
- `accounts/models.py`：UserProfile（角色）、InstanceGroup（实例组）、GroupMembership（成员关系）
- `accounts/views.py`：登录、注册、退出 + 完整 JSON API
- `accounts/urls.py`：路由挂载到 `/accounts/`
- `accounts/management/commands/create_dbsroot.py`：初始化超级用户管理命令

**settings.py 变更**
- 新增 `django.contrib.auth`、`django.contrib.sessions`、`accounts`
- 新增 `SessionMiddleware`、`AuthenticationMiddleware`、`CsrfViewMiddleware`
- REST Framework 改为 `SessionAuthentication` + `IsAuthenticated`
- 新增 `LOGIN_URL = '/accounts/login/'`

**新增模板**
- `templates/accounts/login.html`：登录页
- `templates/accounts/register.html`：注册页（含角色说明提示）

**数据库 / API**
- 执行 `python3 manage.py migrate` 完成建表
- 执行 `python3 manage.py create_dbsroot` 创建初始超级用户
  - 账号：`dbsroot` / 密码：`Dbs@Root2026`

**权限控制逻辑**
- `databases/views.py` 新增 `_can_access_instance()`：query 角色需在实例组中才能访问
- 三个 API 视图（DatabaseListView、TableListView、ExecuteSqlView）均加入鉴权
- query 角色执行 SQL 时后端校验只允许 `SELECT / SHOW / DESCRIBE / DESC / EXPLAIN / USE`

**UI 更新（index.html）**
- 顶栏：显示当前用户名、角色徽章、退出按钮
- 侧边栏：新增「用户管理」「用户组管理」（仅 admin/root 可见）
- 新增「用户管理」面板：查看用户列表、修改角色、删除用户（root 专属）
- 新增「用户组管理」面板：创建组、管理实例、管理成员（模态框操作）
- 所有 fetch 请求加入 `X-CSRFToken` 头和 `credentials: 'same-origin'`
