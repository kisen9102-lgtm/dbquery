## 四、启动服务

**时间**: 中午

### 依赖安装

```bash
cd /opt/dbs_online_v2
pip3 install -r requirements.txt
```

### 修复启动问题

**问题1**：`my_cnf.py` 存在 Python 2 unicode 转义字符串问题
- 报错：`SyntaxError: (unicode error) 'unicodeescape' codec ...`
- 修复：`sed` 修正两处 `\u` / `\h` 转义为 `\\u` / `\\h`

**问题2**：DRF `UNAUTHENTICATED_USER` 依赖 `django.contrib.auth`
- 报错：`RuntimeError: Model class django.contrib.auth.models.Permission doesn't declare an explicit app_label`
- 修复：在 `REST_FRAMEWORK` 配置中添加 `'UNAUTHENTICATED_USER': None`

### 启动命令

```bash
nohup python3 manage.py runserver 0.0.0.0:8000 > logs/runserver.log 2>&1 &
```

**运行状态**：
- 监听地址：`0.0.0.0:8000`
- 日志文件：`/opt/dbs_online_v2/logs/runserver.log`

## 五、浏览器访问

**时间**: 下午

**目标**：在服务器桌面上打开 Chrome 访问服务页面

**排查过程**：
1. 尝试 `DISPLAY=:0` → 失败，X socket 不是 `:0`
2. 检查 `/tmp/.X11-unix/` → 发现 socket 为 `X11`，对应 `DISPLAY=:11`
3. 确认桌面服务为 **xrdp**（端口 3389），非 VNC
4. 使用正确 DISPLAY 启动 Chrome

**最终命令**：

```bash
sudo -u jackhu env \
  DISPLAY=:11 \
  DBUS_SESSION_BUS_ADDRESS="unix:path=/run/user/1000/bus" \
  google-chrome-stable --no-sandbox --new-window \
  "http://127.0.0.1:8000/databases/?ip=127.0.0.1&port=3306&account=test&passwd=test" &
```

**结果**：Chrome 进程成功启动，窗口渲染在 jackhu 的 xrdp 桌面 session（`:11`）中。
**查看方式**：使用 RDP 客户端连接 `服务器IP:3389`，用户名 `jackhu`，即可看到已打开的 Chrome 窗口。
